## 第一章  需求分析

### 1.1 开发背景

随着信息化技术的普及，传统图书馆的人工管理模式逐渐暴露出效率低下、数据查询困难、借阅记录不易追踪等问题。为提升图书管理效率、优化读者借阅体验，开发一套简洁实用的图书管理系统具有现实意义。

本系统模拟真实图书馆管理场景，结合现代化前端交互与后端 Spring Boot 数据管理，打造一个功能完备、操作简便的管理平台。

### 1.2 竞品分析

当前市场上的图书管理系统主要分为两类：一类是大型商业图书馆集成系统（如汇文、金盘等），功能强大但成本高、部署复杂；另一类是开源轻量级系统（如 Koha），但配置繁琐且对中文支持有限。

| 维度 | 本系统 | 商业系统（汇文等） | 开源系统（Koha） |
|------|--------|-------------------|-------------------|
| 部署难度 | 极低（JDK 8 + Maven） | 高（需专业服务器） | 中（需 LAMP 环境） |
| 使用成本 | 免费 | 数万至数十万元 | 免费 |
| 中文支持 | 原生支持 | 良好 | 一般 |
| 界面美观度 | 现代化 UI | 传统桌面风格 | Web 1.0 风格 |
| 二次开发 | 简单（前后端分离） | 困难（闭源） | 较难（Perl） |
| 适用场景 | 小型图书馆/课程设计 | 大型高校图书馆 | 中型图书馆 |

### 1.3 目标用户

- **普通读者**：浏览、搜索、借阅图书，查看借阅记录，管理个人信息
- **系统管理员**：管理图书信息，管理用户状态，监控借阅情况与逾期处理

### 1.4 主要功能

- 用户注册与登录（支持忘记密码重置）
- 图书搜索（关键词、分类）、分页浏览与详情查看
- 图书借阅与归还
- 个人借阅记录查询（按状态筛选）
- 个人信息管理（编辑资料、修改密码）
- **管理端**：图书 CRUD（含封面上传）、用户启用/禁用、借阅记录管理、逾期检查与罚款计算

### 1.5 性能要求

- 单页面加载时间 < 3 秒
- 支持并发借阅操作
- H2 文件数据库自动初始化与种子数据填充，零外部依赖
- 后端启动即完成数据库建表与种子数据填充

**表结构详情**：

**user（用户表）**

| 字段 | 类型 | 约束 | 说明 |

| password | VARCHAR(100) | NOT NULL | 密码（MD5 加密存储） |

| email | VARCHAR(100) | DEFAULT NULL | 邮箱 |
| role | VARCHAR(20) | NOT NULL, DEFAULT 'USER' | USER / ADMIN |
| status | INT | NOT NULL, DEFAULT 1 | 1=正常 / 0=禁用 |
| create_time | TIMESTAMP | DEFAULT CURRENT_TIMESTAMP | 注册时间 |

**book（图书表）**

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | BIGINT | AUTO_INCREMENT, PK | 图书 ID |
| title | VARCHAR(200) | NOT NULL | 书名 |
| author | VARCHAR(100) | NOT NULL | 作者 |
| isbn | VARCHAR(30) | DEFAULT NULL | ISBN 号 |
| publisher | VARCHAR(100) | DEFAULT NULL | 出版社 |
| category | VARCHAR(50) | DEFAULT NULL | 分类 |
| description | CLOB | DEFAULT NULL | 简介 |
| cover_url | VARCHAR(500) | DEFAULT NULL | 封面 URL |
| total | INT | NOT NULL, DEFAULT 1 | 总数量 |
| available | INT | NOT NULL, DEFAULT 1 | 可借数量 |
| create_time | TIMESTAMP | DEFAULT CURRENT_TIMESTAMP | 入库时间 |
| update_time | TIMESTAMP | DEFAULT CURRENT_TIMESTAMP | 更新时间 |

**borrow_record（借阅记录表）**

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | BIGINT | AUTO_INCREMENT, PK | 记录 ID |
| user_id | BIGINT | NOT NULL | 用户 ID |
| book_id | BIGINT | NOT NULL | 图书 ID |
| borrow_time | TIMESTAMP | NOT NULL | 借阅时间 |
| due_time | TIMESTAMP | NOT NULL | 应还时间（借阅 + 30 天） |
| return_time | TIMESTAMP | DEFAULT NULL | 实际归还时间 |
| status | VARCHAR(20) | NOT NULL, DEFAULT 'BORROWED' | BORROWED / RETURNED / OVERDUE |
| fine_amount | DECIMAL(10,2) | DEFAULT 0.00 | 罚款金额 |
| create_time | TIMESTAMP | DEFAULT CURRENT_TIMESTAMP | 创建时间 |

**设计说明**：
- 密码使用 Hutool `SecureUtil.md5()` 进行 MD5 单向哈希，适用于课程项目，生产环境建议使用 BCrypt
- `available` 字段冗余存储可借数量，避免每次查询时进行 COUNT 计算，提升图书列表查询效率，但需在借阅和归还操作中使用 `@Transactional` 保证数据一致性
- 表结构通过 `classpath:init.sql` 在启动时自动执行（`spring.sql.init.mode: always`）
- 数据库索引覆盖 username、role、title、category、author、user_id、book_id、status 等常用查询字段

**借阅业务规则**（`@Transactional`）：
- 检查图书是否存在且 `available > 0`

- 创建记录：`borrow_time = now`，`due_time = now + 30天`，`status = BORROWED`
- 更新图书：`available = available - 1`
- 归还时仅限状态为 BORROWED 或 OVERDUE 的记录，`available + 1`

### 3.4 关键技术实现

#### 3.4.1 JWT 认证拦截器

系统使用 Spring `HandlerInterceptor` 实现 JWT 认证：

- `JwtInterceptor`：从 `Authorization: Bearer <token>` 请求头提取 JWT，调用 `JwtUtils.verify()` 验证，通过后将 `userId` 和 `role` 注入请求属性（`request.setAttribute`），控制器通过 `@RequestAttribute` 获取当前用户信息。
- `WebMvcConfig`：注册拦截器应用于所有路径，排除 `/api/user/login`、`/api/user/register`、`/api/upload/**`、`/upload/**`。同时配置全局 CORS（允许所有来源）和静态资源映射（`/upload/**` → `file:./upload/`）。
- JWT 密钥和过期时间通过 `@Value` 从 `application.yml` 注入（`jwt.secret`、`jwt.expire`），Token 有效期为 24 小时（86400000ms）。
- Token 生成使用 `com.auth0:java-jwt` 库，HMAC256 算法签名，载荷包含 `userId`、`username`、`role`。

前端对应方案：Axios 请求拦截器自动从 localStorage 读取 token 附加到每个请求头；响应拦截器在收到 401 时自动清空登录态并跳转登录页。Vue Router 的 `beforeEach` 守卫对非公开路由进行登录拦截，对 `meta.role = 'ADMIN'` 的路由进行角色检查。

#### 3.4.2 MyBatis-Plus ORM

系统使用 MyBatis-Plus 3.5.3.1 作为 ORM 框架：

- 实体类使用 Lombok `@Data` 和 MyBatis-Plus `@TableName` 注解
- Mapper 接口继承 `BaseMapper<T>`，自动获得 CRUD 方法，无需 XML 映射文件
- `MyMetaObjectHandler` 自动填充 `createTime`（插入时）和 `updateTime`（插入和更新时）
- 分页查询使用 MyBatis-Plus 的 `Page<T>` 对象和 `LambdaQueryWrapper`
- 配置了 `map-underscore-to-camel-case: true`，自动将数据库下划线字段映射到实体驼峰属性
- SQL 日志通过 `StdOutImpl` 输出到控制台，方便开发调试

#### 3.4.3 H2 嵌入式数据库

- 使用 H2 文件数据库（`jdbc:h2:file:./data/library`），MySQL 兼容模式（`MODE=MySQL`），表名大小写不敏感（`DATABASE_TO_LOWER=TRUE`）
- 每次启动通过 `init.sql` 自动初始化表结构和种子数据（`spring.sql.init.mode: always`）
- H2 Web 控制台在 `/h2-console` 路径可用（开发调试用）
- 数据库文件位于 `./data/library.mv.db`，零外部依赖

#### 3.4.4 文件上传处理

- 使用 Spring Boot 内置的 `MultipartFile` 接收上传文件
- `FileService.upload()` 使用 Hutool `IdUtil.fastSimpleUUID()` 生成唯一文件名，保留原扩展名
- 文件存储到配置的 `upload.path` 目录（默认 `./upload/`）
- 文件访问通过 WebMvcConfig 的静态资源映射 `file:./upload/` 提供 HTTP 访问
- 上传大小限制 10MB（`spring.servlet.multipart.max-file-size`）

#### 3.4.5 逾期罚款计算

管理端手动触发 `POST /api/admin/borrow/check-overdue` 接口，`BorrowService.checkOverdue()` 方法：
1. 查询所有 `status = 'BORROWED'` 且 `due_time < now` 的记录
2. 计算逾期天数（向上取整）
3. 生成罚款金额：`逾期天数 × 0.50 元/天`
4. 更新记录状态为 `OVERDUE`，写入 `fine_amount`

逾期检查采用手动触发模式，未使用 Spring `@Scheduled` 定时任务。

#### 3.4.6 库存一致性维护

借阅（`borrowBook`）和归还（`returnBook`）方法均标注 `@Transactional` 注解，确保在同一个数据库事务中完成：
- 借阅记录的创建/更新
- 图书 `available` 数量的增减

编辑图书总数时，`updateBook` 方法自动重新计算：`available = newTotal - 已借出数量`，防止库存数据错误。

## 第六章  项目总结

### 6.1 项目成果

本项目成功实现了一个功能完整的图书管理系统，涵盖用户认证、图书浏览与搜索、借阅归还、个人信息管理、后台管理等核心功能。系统采用前后端分离架构，前端使用 Vue 3 Composition API 结合 Element Plus 组件库，实现现代化美观的用户界面；后端基于 Spring Boot + MyBatis-Plus + H2 数据库，实现了零外部依赖、开箱即用的运行体验。

### 6.2 技术亮点

1. **H2 嵌入式数据库**：使用 H2 文件数据库（MySQL 兼容模式），数据持久化到本地文件，无需安装任何数据库软件，启动即完成建表和种子数据初始化
2. **MyBatis-Plus 零 XML 开发**：Mapper 接口继承 `BaseMapper<T>` 获得完整 CRUD 能力，配合 `LambdaQueryWrapper` 实现类型安全的动态查询，全程无需编写 SQL 映射文件
3. **Spring 拦截器认证链**：`JwtInterceptor` + `WebMvcConfig` 实现统一的请求拦截和认证处理，JWT Token 无状态认证，支持 24 小时有效期
4. **事务一致性**：借阅和归还操作使用 `@Transactional` 注解，确保借阅记录创建与库存扣减在同一事务中完成
5. **前后端双重权限控制**：后端拦截器验证 Token，前端路由守卫 + Axios 拦截器 + 组件条件渲染形成三层防护
6. **组件化 UI 设计**：登录/注册/忘记密码页采用毛玻璃效果 + 动画粒子 + 品牌面板，卡片悬停动效，提升视觉体验

### 6.3 遇到的困难与解决方案

| 困难 | 解决方案 |
|------|----------|
| H2 数据库语法与 MySQL 不完全兼容 | 配置 `MODE=MySQL` + `DATABASE_TO_LOWER=TRUE`，init.sql 用 H2 兼容语法 |
| 编辑图书时修改总数需同步可用数 | `updateBook` 中自动计算 `newAvailable = newTotal - 已借出数量` |
| 用户重复借阅同一本书 | `borrowBook` 中先查询是否存在未归还记录（BORROWED/OVERDUE），存在则拒绝 |
| 前端图片加载失败无回退展示 | 为每本书维护 `imgFailed` 响应式属性，`@error` 时切换为渐变占位符（显示书名首字符） |
| 忘记密码需分步验证 | 设计两步骤流程：步骤1验证用户名+邮箱，步骤2设置新密码，状态通过 reactive 管理 |

### 6.4 后续升级方向

1. **自动逾期检测**：使用 Spring `@Scheduled` 定时任务替代手动触发
2. **后端角色鉴权**：在 `JwtInterceptor` 或新增 `AdminInterceptor` 中校验 ADMIN 角色，防止越权
3. **邮件通知**：集成 JavaMail，实现注册验证、逾期提醒、密码重置邮件
4. **密码安全增强**：替换 MD5 为 BCrypt（Spring Security 提供 `BCryptPasswordEncoder`）
5. **图书导入导出**：使用 Apache POI 或 EasyExcel 支持 Excel 批量导入
6. **数据统计看板**：集成 ECharts 实现借阅趋势、热门图书、用户活跃度等可视化
7. **Docker 部署**：编写 Dockerfile + docker-compose.yml，实现一键部署
8. **API 文档**：集成 Swagger / Knife4j 自动生成 API 文档
9. **参数校验增强**：使用 `@Valid` + `BindingResult` 统一处理，替代 try-catch RuntimeException
