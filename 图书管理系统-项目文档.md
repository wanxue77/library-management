# 图书管理系统 — 项目文档

## 一、项目概述

基于 **Vue 3 + Node.js + SQLite** 的前后端分离图书管理系统，实现用户借阅与管理端后台两大核心模块。

| 项目信息 | 说明 |
|---------|------|
| 前端框架 | Vue 3 (Composition API) + Vite |
| UI 组件库 | Element Plus |
| 后端框架 | Express.js |
| 数据库 | SQLite (文件数据库，零配置) |
| 认证方案 | JWT (JSON Web Token) |
| 密码加密 | MD5 |

## 六、核心业务流程

### 6.1 借阅流程
```
用户浏览图书 → 点击进入详情 → 点击"立即借阅"
→ 系统检查可借数量 → 创建借阅记录（状态=BORROWED）
→ 可借数量 -1 → 借阅成功
```

### 6.2 归还流程
```
用户进入"我的借阅" → 点击"归还"
→ 系统更新记录（状态=RETURNED，填写归还时间）
→ 可借数量 +1 → 归还成功
```

### 6.3 逾期处理
```
管理员点击"检查逾期"
→ 系统查找 due_time < 当前时间 且 状态=BORROWED 的记录
→ 状态改为 OVERDUE → 计算罚款（逾期天数 × 0.5 元/天）
```

## 七、技术要点

### 7.1 前端
- **Composition API** 编写所有组件
- **Vue Router** 路由懒加载 + `beforeEach` 守卫实现登录拦截和角色权限
- **Axios 拦截器** 自动携带 JWT Token，401 时自动跳转登录页
- **Element Plus** 中文国际化配置
- **Vite 代理** 将 `/api` 和 `/upload` 请求转发到后端

### 7.2 后端
- **Express 中间件** 实现 JWT 认证和角色鉴权（USER / ADMIN 分离）
- **sql.js** 纯 JavaScript SQLite 引擎，无需编译，零配置
- **multer** 处理文件上传，UUID 重命名避免冲突
- 数据文件 `library.db` 首次启动自动创建并初始化种子数据

### 7.3 安全
- 密码 MD5 加密存储
- JWT 24 小时过期
- 管理端接口双重验证（登录 + 角色）
- 文件上传大小限制 10MB
